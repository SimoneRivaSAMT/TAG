Requisti
_______________________

 - Nome: Movimento Giocatore
   Priorita: 1
   Note: L'utente puo' usare dei comandi di tastiera o joystick per muovere il personaggio.
   Sotto requisiti:
    01: Conoscenza base di Unity.

 - Nome: Interfaccia Grafica
   Priorita: 1
   Note: L'utente puo' navigare attraverso l'interfaccia di gioco usando tastiera o joystick.
   Sotto requisiti:
    01: Conoscenza base di Unity.

 - Nome: Ambiente Partita
   Priorita: 1
   Note: Mappa di gioco.
   Sotto requisiti: 
    01: Conoscenza base di Unity.
    02: Conoscenza base di Blender.

 - Nome: Grafica Sito Web
   Priorita: 1
   Note: Grafica HTML, CSS e JavaScript del sito esterno.
   Sotto requisiti:
    01: Conoscenza base di HTML, CSS e JavaScript.

 - Nome: Interazioni Giocatore
   Priorita: 1
   Note: L'utente puo' mirare e puntare un laser ed eventualmente parare i laser degli altri usando i comandi di tastiera o joystick.
   Sotto requisiti:
    01: L'utente riesce a muovere il giocatore.

 - Nome: Impostazioni Interfaccia
   Priorita: 1
   Note: Interfaccia UI per gestire tutte le impostazioni (es: audio, grafica, account, ...).
   Sotto requisiti:
    01: L'interfaccia grafica è utlizzabile.

 - Nome: Ostacoli Casuali
   Priorita: 1
   Note: I muri si generano in maniera casuale all'inizio di ogni partita (algoritmo di Manhattan Mapper).
   Sotto requisiti:
    01: Ambiente di gioco completo.
	02: Conoscenza algoritmo Manhattan Mapper.
	
 - Nome: HUD
   Priorita: 1
   Note: L'Heads-Up Display (HUD) mostrerà varie informazioni durante la partita.
   Sotto requisiti:
    01: Giocatore utilizzabile.
	
 - Nome: IA Nemico
   Priorita: 1
   Note: Un'IA che si comporta come un giocatore.
   Sotto requisiti:
    01: Giocatore funzionante.
	02: Ambiente di gioco.

 - Nome: Database
   Priorita: 1
   Note: Il database conterrà tutte le tabelle utili pero lo storage di dati e per il multiplayer.
   Sotto requisiti:
    01: Conoscenza base di MySQL.
	
 - Nome: Leaderboard Globale
   Priorita: 1
   Note: Sarà una tabella consultabile dal sito web dove verranno mostrati i migliori giocatori di sempre con il punteggio.
   Sotto requisiti:
    01: Database funzionante.
	02: Sito web online.
	
 - Nome: Leaderboard della Partita
   Priorita: 1
   Note: Questa Leaderboard conterrà i punteggi e i giocatori della partita corrente.
   Sotto requisiti:
    01: Giocatore funzionante.
	02: Ambiente di gioco.
		
 - Nome: Multiplayer
   Priorita: 1
   Note: Il giocatore puo' collegarsi ad una sessione con altri giocatori.
   Sotto requisiti:
    01: Giocatore funzionante.
	02: Ambiente di gioco.
	03: Database in funzione.
		
 - Nome: Audio
   Priorita: 1
   Note: Musica e Effetti Sonori del gioco.
   Sotto requisiti:
    01: Giocatore funzionante.
	02: Ambiente di gioco.
		
 - Nome: Modelli ed Animazioni
   Priorita: 1
   Note: I modelli ed animazioni dei personaggi.
   Sotto requisiti:
    01: Giocatore funzionante.
	02: Ambiente di gioco.
	03: Conoscenza base di Blender.
